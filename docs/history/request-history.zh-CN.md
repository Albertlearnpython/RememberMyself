## 2026-03-15 - 第 2 轮

### 需求

继续讨论网站设计，先不开始实现。

补充确认：

- 网站要方便未来新增页面
- 图示要能同时出现在 GitHub 文档和对话里
- 页面原型和数据结构要明确
- 网站内容可公开浏览
- 编辑需要登录
- 书籍和音乐资源也需要登录后访问
- 账号权限需要分层
- 不同页面允许有各自独立风格

### 决策

- 架构必须优先支持未来扩展
- 访问策略为“公开阅读 + 登录编辑”
- 书籍和音乐文件采用“登录后访问”
- 权限角色至少包含 `owner / editor / viewer`
- 视觉系统允许页面独立，而不是强行统一成一个模板

### 影响页面

- 全局架构
- 登录与权限系统
- 书籍
- 音乐
- 全站样式策略

规划已更新。
