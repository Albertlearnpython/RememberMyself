# 喜欢的美食最终视觉稿说明

## 目标

这份文档定义“喜欢的美食”页面的最终视觉规格。

这一页要同时成立三件事：

- 像料理档案
- 像有温度的生活收藏
- 又足够清晰，方便后续做增删改查

它不是外卖列表，也不是菜谱网站首页。

## 页面路由

- 美食首页：`/food`
- 美食详情高亮：`/food/:id`
- 编辑态：保留在 `/food`，通过抽屉打开

## 页面视觉基调

关键词：

- 温暖
- 克制
- 食物的质感
- 柜台卡片感
- 有点手工料理笔记的感觉

## 页面布局

### 桌面端

- 页面主容器：`1240px`
- 左侧筛选：`220px`
- 中间列表：`1fr`
- 右侧详情：`400px`
- 栏间距：`24px`

### 手机端

- 左右留白：`16px`
- 列表单列
- 详情改全屏
- 编辑用全屏页或底部抽屉

## 头部工具区

### 结构

- 左：页面标题
- 中：搜索框
- 右：新增美食

### 搜索框

- 高度：`42px`
- 圆角：`12px`
- 背景：`rgba(251,245,238,0.92)`
- 边框：`1px solid var(--rm-food-line)`

### 新增按钮

- 高度：`42px`
- 背景：`var(--rm-food-accent-clay)`
- 文字：`#FBF5EE`
- 圆角：`10px`

## 左侧筛选栏

### 推荐筛选

- 类型：家常 / 面食 / 甜品 / 饮品 / 夜宵
- 心情：安慰 / 清爽 / 满足 / 节制
- 是否含图
- 是否有详细做法

### 样式

- 筛选 chip 高度：`32px`
- 圆角：`999px`
- 选中态背景：`var(--rm-food-accent-soft)`

## 中间美食列表

### 列表样式

- 第一版采用双列卡片
- 卡片间距：`18px`
- 卡片最小高度：`260px`

### 单张卡片结构

1. 顶部食物照片
2. 菜名
3. 两到三个标签
4. 一句说明
5. 轻量进入提示

### 卡片尺寸

- 图片高度：`156px`
- 卡片内边距：`16px`
- 圆角：`18px`
- 阴影：`var(--rm-food-shadow-soft)`
- hover：上移 `2px`

### 图片规则

- 比例：`4:3`
- 圆角：`14px`
- 填充方式：`cover`

## 右侧详情区

### 内容顺序

1. 大图
2. 菜名
3. 标签 / 场景 / 更新时间
4. 为什么喜欢
5. 做法内容
6. 附加备注

### 重点区块：为什么喜欢

这是美食页区别于普通菜谱页的关键。

样式建议：

- 左侧 `2px` 竖线
- 正文 `16px / 1.85 / 500`
- 颜色略深于普通正文

### 做法区

- 保持段落阅读感
- 不做强表格
- 支持 Markdown 风格小标题和列表

## 编辑抽屉

### 分区

1. 基础信息
2. 图片上传
3. 喜欢原因
4. 做法与备注

### 字段建议

- 名称
- 标签
- 一句摘要
- 图片
- 为什么喜欢
- 做法
- 备注
- 可见性

### 抽屉样式

- 宽度：`520px`
- 顶部固定标题
- 底部固定保存区
- 输入框高度：`42px`
- 多行输入最小高度：`120px`

## 手机端规则

- 美食卡片改单列
- 图片优先于文字
- 详情页顶部保留返回与编辑
- 做法区必须可顺滑纵向阅读

## 前端实现验收标准

- 第一眼像美食收藏，不像菜谱平台
- 图片必须有质感，但不能做满屏大图站
- “为什么喜欢”必须成为页面个性核心
- 卡片和详情都要有温度，不要后台感
