# 首页版式深化说明

## 当前版式目标

首页现在已经不是“先讨论方向”的阶段，而是进入了可直接照着做的版式阶段。

这一版主要解决两个问题：

- 第一屏空白过大
- 首页结构太散，没有稳定骨架

## 当前桌面端栅格

- 页面最大宽度：`1520px`
- 主内容双栏：`360px + auto`
- 模块索引：`3 列`
- 最近更新：`4 列`

## 当前结构顺序

```text
顶部导航
→ 首屏双栏（个人档案卡 / 介绍主卡）
→ 模块索引
→ 五条记忆溪流
→ 最近更新
→ 页尾摘录
```

## 首屏版式细则

### 左栏：个人档案卡

建议宽度：

- `280px ~ 360px`

内部顺序：

1. 头像 + 姓名
2. 地点 / 学校
3. 标签
4. 时间线
5. 联系方式

这张卡在桌面端保持 `sticky`，用于稳定整页的左侧重心。

### 右栏：介绍主卡

内容顺序：

1. 眉标
2. 小彩点
3. 大标题
4. 副说明
5. Motto
6. 按钮组
7. 统计卡 3 张

要求：

- 不能再有大面积无效留白
- 右栏要足够饱满，但仍保持留白

## 模块索引版式

- 桌面端 `3 列`
- 每张整卡可点
- 卡片高度基本一致
- 负责秩序，不负责抢首页重心

## 五条溪流的嵌入关系

### 为什么把流带放在模块索引后面

因为首页需要先给出结构，再给出情绪。

顺序固定为：

1. 先知道网站有什么
2. 再看到内容怎样流动

这样首页不会失去导航性。

## 流带版式规则

每条流带都由两层组成：

- 上层：标题 + 说明 + 数量 + 入口
- 下层：横向流动视窗

容器统一纵向堆叠，避免做成横向四宫格。

## 最近更新区版式

- 桌面端 `4 列`
- 卡片比模块索引更轻
- 不承担主视觉，只证明网站还在生长

## 手机端收缩规则

### 860px 以下

- 首屏改为单列
- 个人档案卡不再 sticky
- 模块索引改为单列
- 统计卡改为单列
- 流带标题区变上下排列

### 手机端核心原则

- 一屏必须能看到首页的“人物信息”
- 一屏必须能看到至少 1 个明确动作按钮
- 不允许只剩漂亮背景，没有有效信息

## 当前版式结论

首页版式已经正式定为：

**信息先稳住，情绪再流出来。**

也就是：

- 双栏首屏负责“立住人”
- 模块索引负责“立住结构”
- 记忆溪流负责“立住气质”
