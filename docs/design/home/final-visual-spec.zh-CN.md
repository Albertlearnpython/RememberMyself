# 首页最终视觉稿说明

## 目标

这份文档是首页的最终视觉规格说明。

它不是继续讨论方向，而是给前端一份可以直接照着实现的页面标准。

目标：

- 定死首页视觉 token
- 定死页面层级和尺寸
- 定死首页点击跳转逻辑
- 定死桌面端和手机端的交互表现

## 首页路由与点击跳转

这一部分是第一版必须明确实现的内容。

### 首页主路由

- 首页：`/`

### 各板块路由

- 收藏书籍：`/books`
- 喜欢的美食：`/food`
- 喜欢的音乐：`/music`
- 喜欢的景色：`/scenery`
- 健身养体：`/fitness`
- 收支平衡：`/finance`
- 时间安排：`/schedule`
- 方法心得：`/methods`

### 首页内部锚点

- 当前关注区：`/#current-focus`
- 模块索引区：`/#module-index`
- 最近更新区：`/#recent-updates`

### 首页可点击区域

| 区域 | 点击行为 | 桌面端 | 手机端 |
| --- | --- | --- | --- |
| 站点名 `RememberMyself` | 返回首页 `/` | 是 | 是 |
| 顶栏 `模块` | 打开模块面板 | 下拉面板 | 底部弹层 |
| 模块面板内任一板块 | 跳转对应板块路由 | 是 | 是 |
| 顶栏 `归档` | 滚动到最近更新区 `/#recent-updates` | 是 | 是 |
| Hero 主按钮 | 跳转 `/books` | 是 | 是 |
| Hero 次按钮 | 滚动到 `/#current-focus` | 是 | 是 |
| 当前关注中的模块标签 | 跳转对应模块页 | 是 | 是 |
| 模块索引卡片整卡 | 跳转对应模块页 | 是 | 是 |
| 最近更新卡片 | 跳转对应模块页，并高亮相关记录 | 是 | 是 |

### 模块入口规则

首页进入别的板块，不只靠一个地方。

第一版确定为四层入口：

1. 顶栏 `模块`
2. Hero 主按钮
3. 模块索引卡片
4. 最近更新卡片

这四层里，真正的主入口是：

- `模块索引卡片`

因为它最稳定，也最像“人生系统总索引”。

## 页面容器与栅格

### 桌面端

- 页面最大宽度：`1440px`
- 内容主容器：`1120px`
- 左右安全留白：最少 `56px`
- 栅格：`12 columns`
- 栏间距：`24px`

### 手机端

- 内容宽度：`calc(100vw - 32px)`
- 左右留白：`16px`
- 纵向区块间距：`20px`

## 区块视觉说明

## 1. 顶栏

### 结构

- 左：站点名
- 中：`模块`
- 右：`归档`、`登录`

### 样式

- 背景：透明
- 底部不加重线，只有极淡 1px 边线
- 站点名字号：`18px`
- 站点名字重：`600`
- 导航名字号：`13px`
- 导航名字重：`500`
- 字间距：`0.02em`

### 点击反馈

- hover：文字变为 `--rm-home-accent-moss`
- active：透明深色底 `rgba(31,29,26,0.04)`

### 模块面板

桌面端：

- 由 `模块` 按钮触发下拉面板
- 面板宽度：`320px`
- 面板内每一项高度：`52px`
- 每一项左侧是板块名，右侧是极轻箭头

手机端：

- 点击 `模块` 打开底部弹层
- 弹层高度：屏高 `72%`
- 每项整行可点

## 2. Hero

### 内容规则

- 日期与城市放顶部
- 主文案只允许一行或两行
- 辅助文案控制在 3 行以内
- 操作按钮固定两个

### 样式

- 背景不做实心大卡片
- 只在内容外层使用极轻边框区隔
- 主文案颜色：`--rm-home-text-strong`
- 辅助文案颜色：`--rm-home-text-body`
- 元信息颜色：`--rm-home-text-muted`

### 按钮样式

主按钮：

- 高度：`44px`
- 内边距：`0 18px`
- 背景：`var(--rm-home-accent-moss)`
- 文字：`#F4EFE6`
- 边框：`1px solid var(--rm-home-accent-moss)`
- 圆角：`10px`

次按钮：

- 高度：`44px`
- 内边距：`0 18px`
- 背景：`transparent`
- 文字：`var(--rm-home-text-strong)`
- 边框：`1px solid var(--rm-home-line)`
- 圆角：`10px`

按钮 hover：

- 主按钮：亮度提高 `4%`
- 次按钮：背景变为 `rgba(31,29,26,0.03)`

## 3. 当前关注区

### 布局

- desktop：四等分横排
- mobile：两列两行

### 单项样式

- 上：小标签 `12px / 500`
- 中：当前状态 `16px / 500`
- 下：更新时间 `12px / 500`
- 区块内边距：`20px`
- 外边框：`1px solid var(--rm-home-line)`
- 背景：`rgba(248,244,238,0.6)`

### 点击规则

每一项都可以点击进入对应板块。

例如：

- 当前阅读 -> `/books`
- 当前身体目标 -> `/fitness`
- 本周收支 -> `/finance`
- 最近方法 -> `/methods`

## 4. 模块索引区

### 布局

desktop：

- `2 列 x 4 行`
- 列间距：`24px`
- 行间距：`20px`

mobile：

- 单列
- 卡片间距：`12px`

### 模块卡片

每张卡片结构：

- 上：模块名
- 中：一句摘要
- 下：极轻的进入提示 `进入`

### 卡片样式

- 最小高度：`124px`
- 内边距：`22px`
- 背景：`var(--rm-home-surface)`
- 边框：`1px solid var(--rm-home-line)`
- 圆角：`14px`
- 阴影：`var(--rm-home-shadow-soft)`

### 卡片交互

- 整卡可点
- hover：上移 `2px`
- hover：边框改为 `rgba(70,82,70,0.26)`
- hover：阴影变为 `var(--rm-home-shadow-hover)`
- active：缩回并降低亮度

这是首页进入其他板块的核心跳转方式。

## 5. 最近更新区

### 布局

desktop：

- 4 列
- 每卡等高

mobile：

- 单列
- 保留 4 张纵向卡片

### 卡片样式

- 最小高度：`168px`
- 内边距：`18px`
- 背景：`rgba(248,244,238,0.82)`
- 边框：`1px solid var(--rm-home-line)`
- 圆角：`14px`

### 卡片内容顺序

1. 模块名
2. 一句最近摘要
3. 时间
4. 进入提示

### 点击逻辑

- 点击整卡跳到对应模块页
- 若模块页已有详情页能力，则默认高亮最近那条记录
- 第一版若无详情高亮，就先跳到对应模块主列表页

## 6. 页脚摘录

### 样式

- 字号：`18px`
- 行高：`1.8`
- 字重：`400`
- 使用标题字体
- 颜色：`--rm-home-text-body`
- 文本宽度不超过 `32ch`

这个区域不做按钮，不做复杂交互。

## 动效规范

- 页面进入：区块 stagger fade-in，单个区块延迟 `40ms`
- 卡片 hover：`160ms ease`
- 面板/弹层打开：`220ms cubic-bezier(0.22, 1, 0.36, 1)`
- 不使用 bounce
- 不使用大位移动画

## 手机端具体规则

### 顶栏

- 保留站点名、`模块`、`登录`
- `归档` 收进 `模块` 弹层底部

### Hero

- 主文案字号降到 `34px`
- 按钮允许换行，不强制一行摆下

### 当前关注

- 改成两列网格

### 模块索引

- 单列整卡
- 每张卡片右侧保留轻箭头

## 最终验收标准

- 一进入首页，能看懂这是“人生系统总索引”
- 不靠看说明，也能找到进入其他板块的方式
- 首页至少存在 4 层跳转入口
- 模块索引卡片必须整卡可点
- 手机端也必须能快速进入任一板块

## 本版结论

之前首页确实有“模块索引”的概念，但没有被写成正式点击规则。

这一版已经补全：

- 从首页点击跳转到其他板块的方式
- 每个入口的行为
- 桌面端和手机端的展开方式
- 前端可直接实现的视觉尺寸与交互状态
