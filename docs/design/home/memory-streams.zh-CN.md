# 首页四条流带设计稿

## 当前定位

四条流带不是模块宫格，也不是内容列表。

它们现在的定位是：

**首页中的四条缓慢内容河道。**

作用：

- 让首页出现真实内容的气味
- 让不同模块提前“发声”
- 把静态首页变成会流动的总索引

## 当前四条流带

| 流带 | 数据状态 | 当前颜色 |
| --- | --- | --- |
| 书影流 | 已接真实书籍封面 | 蓝 |
| 声纹流 | 已预留结构，待真实音乐 | 紫 |
| 食味流 | 已预留结构，待真实美食 | 橙 |
| 风景流 | 已预留结构，待真实景色 | 绿 |

## 当前卡面材质

### 流带容器

- 浅色珍珠面
- 顶部细线轻高亮
- 底层带主色光斑

### 流带卡片

- 深色矿石面
- 顶部有对应流带色的光晕
- 底部有深色文字遮罩
- 外层有一圈轻微高光边

### hover

- 卡片上浮 `2px ~ 4px`
- 封面轻微放大
- 边线更亮

## 当前各流带气质

### 书影流

- 主色：蓝
- 感觉：理性、整齐、归档
- 文字：书名 + 阅读状态

### 声纹流

- 主色：紫
- 感觉：夜色、回声、余韵
- 后续真实化后：专辑封面 + 标题

### 食味流

- 主色：橙
- 感觉：温度、生活、烟火
- 后续真实化后：食物照片 + 名称

### 风景流

- 主色：绿
- 感觉：空气、距离、远景
- 后续真实化后：景色照片 + 地点

## 当前动效规则

- 书影流 / 食味流默认向左
- 声纹流 / 风景流默认向右
- 支持 `1 - 2` 条平行流道
- 单条时长保持在 `58s - 92s`
- hover 整条流带时暂停动画

## 当前占位卡规则

- 使用深色虚线边框
- 保留轻微扫光
- 不伪造真实用户内容

## 当前设计目的

首页里的流带不是为了“多一个展示组件”。

真正的目的，是让你以后把：

- 全部书籍
- 全部音乐
- 全部美食
- 全部景色

都以一种像溪流那样的方式挂到首页。

## 当前结论

四条流带已经正式成为首页的固定结构，而不是临时设计点子。

以后新增真实数据，只需要接入数据，不需要推翻这套外壳。
