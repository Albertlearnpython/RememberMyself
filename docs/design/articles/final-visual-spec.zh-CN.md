# 文章板块最终视觉稿说明

## 当前定位

文章板块用于管理 Markdown 文档。

第一版职责收敛为：

- 上传 `.md`
- 下载 `.md`
- 编辑正文
- 保存时覆盖源文件
- 直接在页面渲染 Markdown
- 接入首页“字句流”

## 页面气质

关键词：

- 安静
- 清楚
- 像文字档案室
- 有编辑台感，但不是博客后台
- 正文可长时间阅读，但页面本身不过度装饰

## 页面结构

桌面端三栏：

- 左：筛选与说明
- 中：文章列表
- 右：文章详情与 Markdown 渲染区

手机端：

- 单列
- 列表优先
- 详情在当前页面下方继续展开
- 编辑依旧使用抽屉全屏覆盖

## 页面头部

结构：

- 左：`文章板块 / 文字档案室`
- 中：搜索框
- 右：新增文章按钮

搜索范围：

- 标题
- 摘要
- 正文
- 文件名

## 左侧筛选区

第一版只保留：

- 可见性筛选
- 模块说明

不在第一版做：

- 标签
- 分类树
- 多级目录

## 中间列表卡

### 卡片结构

- 左：`MD` 文件封面片
- 中：标题、文件名、摘要、可见性、文件大小
- 右：更新时间、查看提示

### 卡片规格

- 外层圆角：`16px`
- 左侧文件封面片：`84px x 101px`
- 内边距：`18px`
- hover：轻微上浮 + 蓝紫边界提亮
- 选中态：边框更亮，底色更偏冷白

## 右侧详情区

内容顺序：

1. 标题与摘要
2. 文件信息
3. 下载入口
4. Markdown 渲染正文
5. 删除操作

## Markdown 正文区

要求：

- 标题使用衬线字
- 正文维持舒展行距
- 引用块、代码块、表格都要有清晰样式
- 不做阅读器式仿纸背景
- 保持冷白底上的高级多彩细节

## 编辑抽屉

抽屉分两段：

1. 源文件与基础信息
2. Markdown 正文

关键提示要明确：

- 上传新文件会覆盖正文
- 直接改正文后保存，也会覆盖服务器上的 `.md`

## 首页“字句流”

首页新增第五条流带：

- 名称：`字句流`
- 数据源：文章模块真实数据
- 卡片类型：无封面时用文字首字兜底
- 点击行为：进入文章详情

气质要求：

- 不做图片流
- 更像文稿索引卡缓慢经过
- 与书影流、声纹流并列，但更偏文字感

## 第一版验收标准

- 能上传 Markdown 文件
- 能在页面直接渲染 Markdown
- 能下载当前最新源文件
- 编辑保存后，下载到的是覆盖后的最新版本
- 首页出现真实“字句流”
