# 全站色彩系统方案

## 当前方案名

`流彩索引 2.0`

这是 `RememberMyself` 当前正在落地的正式配色，不再是早期那种偏米黄、偏纸张、偏阅读器的路线。

这次方案的目标只有一句话：

**五彩斑斓，但一定要贵。**

## 这次要解决的问题

- 首页不能再有“淡黄色阅读器”的既视感
- 页面不能只有一种颜色，不然会显得单薄、廉价、没层次
- 多彩不能变成乱彩，必须保持秩序、留白和节制

所以现在的策略不是“把页面刷花”，而是建立一套更像珠宝 / 矿石 / 玻璃的色彩层级：

- 底：冷白蓝灰
- 骨：深蓝黑
- 光：青 / 蓝 / 紫 / 橙 / 粉 / 绿
- 点睛：少量金色

## 总体配比

- `55%` 冷白与蓝灰底色
- `25%` 深色骨架与深色锚点
- `20%` 蓝 / 青 / 紫 / 橙 / 粉 / 绿的彩光点缀

重点不是高饱和，而是：

- 有色相变化
- 有深浅层级
- 有主次秩序

## 核心原则

### 1. 浅色背景必须冷，不再偏黄

全站大底已经切换为冷白蓝灰，不再回到暖象牙白。

因为一旦暖黄占据大面积背景，页面就会重新滑回：

- 阅读器感
- 模板感
- 年代感

### 2. 多彩只出现在“光”和“边缘”，不大面积铺满

高级的多彩，不是整块背景高饱和。

应该出现在这些位置：

- Hero 的极光层
- 模块卡底部或边缘的彩光
- 流带容器的轻微偏色
- 流带卡片的矿石高光
- 标签 / 按钮 / 状态的小面积强调

### 3. 必须保留深色锚点

没有深色锚点，多彩会显得浮。

当前深色锚点固定在：

- 首页右侧介绍主卡
- 五条流带中的卡片本体
- 主按钮
- 少量统计卡

### 4. 内容封面永远比配色更重要

书封、专辑封面、照片才是真主角。

配色系统只负责：

- 把场子搭对
- 把层级拉开
- 把情绪托起来

## 首页五条溪流配色

| 流带 | 主色 | 卡片气质 |
| --- | --- | --- |
| 书影流 | `#5C87FF` | 冷静、理性、书封归档 |
| 字句流 | `#4F7CFF` | 文字索引、蓝紫过渡、克制发光 |
| 声纹流 | `#8B5CF6` | 夜色、回声、层次感 |
| 食味流 | `#FF8F3E` | 温度、烟火、轻暖感 |
| 风景流 | `#17A874` | 空气、远景、呼吸感 |

规则：

- 流带外壳是浅色珍珠面
- 流带卡片本体是深色矿石面
- 色彩主要出现在边缘光、顶部线、卡片高光和封面互动里

## 首页当前落地规则

### 首屏

- 左侧个人档案卡：浅色玻璃面 + 轻彩光
- 右侧介绍主卡：深色极光面 + 多色光晕
- 统计卡：深色底，但每张有不同的彩色顶线

### 模块入口

- 保持浅色卡片
- 用底角彩光和弱渐变体现模块区别
- 不做大块彩底，避免像工具站首页

### 记忆溪流

- 流带容器：浅色、通透、带一点珍珠质感
- 流带卡片：深色矿石感、镜面高光、底部深色遮罩
- hover：轻微上浮、封面微放大、边线发亮

## 组件级限制

### 禁止项

- 禁止大面积暖黄背景
- 禁止荧光绿 / 荧光粉标签
- 禁止全站只有一种主色
- 禁止把所有卡片都做成一个颜色

### 允许项

- 允许彩色光晕
- 允许玻璃质感
- 允许深色卡片与浅色页面并存
- 允许不同模块有轻微独立气质

## 当前已落地到哪些地方

- 全站基础 token
- 顶栏玻璃感
- 首页个人档案卡
- 首页介绍主卡
- 首页统计卡
- 首页模块入口卡
- 五条记忆溪流容器与卡面

## 结论

`RememberMyself` 现在的色彩路线已经不是“温柔米黄”，而是：

**冷白为底，深色为骨，多彩为光。**

它要看起来像一本被认真装帧过、同时又带一点现代玻璃与矿石质感的个人档案册。
