# 全站色彩系统方案

## 方案名

流彩索引。

这套系统服务于整个 `RememberMyself`，不是某一个页面的临时修色。

这次方案在上一版基础上继续推进，明确解决三个问题：

- 网站不能再像“淡黄读书阅读器”
- 网站也不能退回“整站深色科技感”
- 网站需要真正具备“多彩但不俗”的个人气质

这次真正落地的方向是：

- 以冷白和浅蓝灰做底
- 用蓝、青、紫、橙、粉做多点缀结构
- 让不同页面与模块天然带轻微彩度，而不是只有一两种颜色

这套思路参考了用户提供的东方色谱 / 国家级配色灵感，但只吸收“配色结构”，不直接照搬国风色卡。

## 总体配比

- `50%` 冷白与浅蓝灰底色
- `30%` 深蓝骨架与夜色锚点
- `20%` 蓝 / 青 / 紫 / 橙 / 粉的多彩强调

核心目标不是“更花”，而是“更贵”。

## 核心原则

### 1. 暖色只能点睛，不能统治页面

铜棕和偏暖色只允许出现在：

- 下载 / 资源提示
- 局部插画渐变
- 模块气质点缀

它们不能再大面积铺成背景，不然页面会直接滑回“米黄阅读器感”。

### 2. 整站必须有一个深色锚点

深色锚点不是全站深色，而是出现在这些位置：

- 首页 Hero
- 溪流卡片
- 主按钮
- 少量强调容器

这样网站才有骨架，不会变成“全浅色、全松散、全没重心”。

### 3. 内容优先于配色

书封、专辑封面、食物照片、景色照片才是真正的主角。

配色系统负责：

- 结构秩序
- 情绪氛围
- 信息层级

它不负责抢内容的视觉中心。

### 4. 对比要克制，但不能发虚

高级感不是“全都很淡”。

必须同时保证：

- 背景够安静
- 文字够清楚
- 卡片边界够明确
- 主次关系一眼能分出来

## 首页四条溪流配色

| 流带 | 主色 | 卡片气质 |
| --- | --- | --- |
| 书影流 | `#70879D` | 冷静、理性、归档感 |
| 声纹流 | `#6F6886` | 夜晚、回声、余韵 |
| 食味流 | `#91664A` | 温度点到即止 |
| 风景流 | `#58796F` | 空气感、远景感 |

规则：

- 四条流带都在同一套瓷白背景上成立
- 差异来自“偏色”和“深浅关系”，不是高饱和抢眼
- 每条流带底下都允许出现深色卡片，作为节奏锚点

## 组件级落地规则

### 页面背景

- 整站底色统一使用冷瓷灰，不允许再回到大面积暖黄
- 背景必须带轻微层次，避免像后台纯白页面

### 顶栏

- 顶栏是半透明瓷白，不是实心米黄
- hover 态改为偏墨灰轻覆盖，而不是偏暖色发糊

### 卡片

- 主卡片背景统一是高层瓷白
- 卡片靠细边框和轻阴影建立层次
- 悬浮态优先提边界和阴影，不靠大幅变色

### Hero 与溪流卡片

- Hero 允许成为全站主要深色锚点
- 溪流卡片保持深色矿石感，让内容封面更稳
- 深色只集中在局部，不扩大到整个页面

### 按钮

- 主按钮使用墨蓝渐变，不再使用偏暖绿色底
- 次按钮用高层浅底 + 细边框
- 危险按钮统一归到漆红体系

### 标签

- 标签继续允许不同颜色
- 但全部压低饱和度，避免荧光标签
- 默认标签底改为浅表面，而不是深底胶囊

### 抽屉与弹窗

- 抽屉、弹窗、批量补全面板全部收口到同一套瓷白层级
- 背景遮罩明显加深，让浮层真正浮起来
- 内部表单卡片继续使用高层浅底，不做发黄纸面

## 当前实施范围

本轮已经实际落地到：

- 全站基础 token
- 首页 Hero 与四条溪流
- 模块索引与最近更新
- 收藏书籍页
- 喜欢的音乐页
- 编辑抽屉、资料补全弹窗、批量补全面板、toast

## 结论

这套色彩系统不再追求“温柔就等于高级”。

真正确定下来的，是下面这组关系：

- 底：瓷白
- 骨：墨色
- 节奏：深色锚点
- 点睛：玉青 / 雾蓝 / 铜棕 / 漆红

最终目标是让网站像一本被认真装帧过的个人档案册，而不是阅读器模板，也不是通用后台。
