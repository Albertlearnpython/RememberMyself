# 方法心得初步设计说明

## 页面定位

方法心得页不是普通博客文章列表。

它更像：

- 个人方法论档案
- 一些长期有效原则的保存页
- 可反复回看的思考集合

## 页面路由

- `/methods`

正式实现时建议使用：`/methods`

## 结构建议

### 桌面端

- 左：标签与分类
- 中：文章列表
- 右：文章详情

### 手机端

- 顶部搜索与标签
- 文章列表
- 点击进入详情

## 核心字段

- 标题
- 内容
- 标签
- 创建时间
- 更新时间
- 一句摘要

## 视觉基调

- 安静
- 有思想密度
- 接近纸质阅读
- 不做媒体站信息流

## 关键交互

- 列表与详情联动
- Markdown 渲染
- 标签筛选

## 后续深化方向

- 文章排版规格
- 标签系统
- 首页摘录如何引用方法页内容
