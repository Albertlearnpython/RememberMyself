# 喜欢的音乐最终视觉稿说明

## 目标

这份文档定义“喜欢的音乐”页面的最终视觉规格。

这个页面要同时处理三种体验：

- 音乐收藏
- 在线播放
- 受保护资源访问

它要像私人听音室，不像网盘，也不像通用播放器。

## 页面路由

- 音乐首页：`/music`
- 曲目详情高亮：`/music/:id`
- 编辑态：保留在 `/music`，通过抽屉打开

## 页面视觉基调

关键词：

- 深一点，但不阴沉
- 有唱片柜感
- 安静
- 有节奏感
- 有夜晚听歌的质地

## 页面布局

### 桌面端

- 左侧筛选：`220px`
- 中间曲库列表：`1fr`
- 右侧详情与播放器：`400px`
- 栏间距：`24px`

### 手机端

- 单列曲目卡片
- 点击后进入全屏详情与播放器
- 编辑采用全屏页或底部抽屉

## 头部工具区

### 结构

- 左：页面标题
- 中：搜索框
- 右：新增音乐

### 搜索框

- 高度：`42px`
- 背景：`rgba(246,241,235,0.92)`
- 边框：`1px solid var(--rm-music-line)`
- 圆角：`12px`

### 新增按钮

- 背景：`var(--rm-music-accent-vinyl)`
- 文字：`#F6F1EB`
- 高度：`42px`
- 圆角：`10px`

## 左侧筛选栏

### 推荐筛选

- 类型：流行 / 纯音乐 / 古典 / 摇滚 / 电子 / 轻音乐
- 情绪：安静 / 清醒 / 低落 / 鼓舞 / 夜晚
- 是否可播放
- 是否可下载

### 样式

- 筛选 chip：`32px`
- 选中背景：`var(--rm-music-accent-soft)`

## 中间曲库列表

### 排布

- 第一版采用纵向列表
- 每项最小高度：`112px`
- 项间距：`12px`

### 单项结构

左：

- 封面

中：

- 曲名
- 歌手 / 专辑
- 标签
- 一句说明

右：

- 时长
- 是否可播放
- 轻箭头

### 卡片样式

- 背景：`var(--rm-music-surface)`
- 边框：`1px solid var(--rm-music-line)`
- 圆角：`16px`
- 内边距：`16px`
- 封面：`72px x 72px`
- hover：上移 `2px`

## 右侧详情与播放器区

### 内容顺序

1. 封面
2. 曲名
3. 歌手 / 专辑 / 时长
4. 为什么喜欢
5. 播放器
6. 文件区
7. 附加备注

### 播放器区

这是音乐页最重要的功能区。

建议结构：

- 当前进度条
- 播放 / 暂停
- 上一首 / 下一首
- 当前时间 / 总时长

### 播放器样式

- 外层背景：`rgba(237,230,221,0.65)`
- 圆角：`14px`
- 内边距：`16px`
- 进度条高度：`4px`
- 已播放颜色：`var(--rm-music-accent-vinyl)`

## 受保护文件区

### 游客态

- 能看到文件存在
- 能看到格式和时长
- 不能播放和下载
- 按钮灰化

### Viewer 态

- 可在线播放
- 可下载

### Editor / Owner 态

- 增加替换文件、修改可见性、删除

## 编辑抽屉

### 分区

1. 基础信息
2. 封面与标签
3. 为什么喜欢
4. 文件与权限

### 字段建议

- 曲名
- 歌手
- 专辑
- 时长
- 标签
- 一句说明
- 为什么喜欢
- 音频文件
- 封面图
- 是否允许播放
- 是否允许下载
- 可见性

### 样式

- 抽屉宽度：`520px`
- 输入框高度：`42px`
- 多行输入：`120px`

## 手机端规则

- 曲目卡片改单列
- 详情页上半是封面与标题，下半是播放器与文件
- 播放器按钮要单手可操作
- 时间信息用等宽字体

## 前端实现验收标准

- 页面第一眼像私人音乐收藏，不像 SaaS 播放器
- 播放器简洁，不堆复杂按钮
- 游客态的锁定要克制，不要像报错
- 列表和详情都要有“听音室”的安静感
