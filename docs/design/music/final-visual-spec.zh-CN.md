# 喜欢的音乐最终视觉稿说明

## 当前定位

音乐板块这一版不做在线播放。

第一版正式定位为：

**音乐收藏与归档页**

它负责：

- 上传音乐文件
- 下载音乐文件
- 展示封面与文字信息
- 记录为什么喜欢
- 接入首页“声纹流”

它不负责：

- 在线播放器
- 进度条
- 上一首 / 下一首
- 歌词滚动

## 页面气质

关键词：

- 心灵的极简主义
- 像私人听音档案柜
- 夜晚，但不阴沉
- 安静，有留白
- 有音乐气质，但不是播放器产品页

## 页面路由

- 音乐首页：`/music`
- 音乐详情高亮：`/music/:id`
- 编辑态：仍在 `/music` 中，以抽屉打开

## 页面布局

### 桌面端

- 左侧筛选：`220px`
- 中间音乐列表：`1fr`
- 右侧详情面板：`380px`
- 间距：`24px`

### 手机端

- 单列音乐卡片
- 详情区下沉到列表下方
- 编辑保持抽屉或全屏覆盖层

## 页面头部

结构：

- 左：页面标题
- 中：搜索框
- 右：新增音乐

要求：

- 搜索框和书籍页保持一致的秩序感
- 按钮用全站主按钮样式
- 不出现播放器入口按钮

## 左侧筛选

第一版只保留简单筛选：

- 标签
- 可见性

不做：

- 风格分类树
- 情绪播放器筛选
- 是否可播放

## 中间音乐列表

### 布局

- 纵向列表
- 每项最小高度：`112px`
- 项间距：`12px`

### 单项结构

左：

- 方形封面

中：

- 曲名
- 歌手 / 专辑
- 标签
- 一句短评

右：

- 更新时间
- 查看提示

### 卡片样式

- 背景：浅卡片
- 边框：细边框
- 圆角：`16px`
- 封面尺寸：`72px x 72px`
- hover：轻微上浮

## 右侧详情区

内容顺序：

1. 方形封面
2. 曲名
3. 歌手 / 专辑
4. 标签
5. 一句短评
6. 为什么喜欢
7. 长说明
8. 音乐文件下载区

## 文件区

音乐文件区的逻辑和书籍附件区一致，但文案要更贴合音乐。

### 游客态

- 能看到存在音乐文件
- 能看到文件名和大小
- 不能下载

### 登录用户

- 如果文件可下载，则允许下载
- 如果权限不足，则显示克制的不可下载提示

### 编辑者

- 可上传文件
- 可删除文件
- 可修改音乐资料

## 编辑抽屉

分区：

1. 基础信息
2. 封面与标签
3. 为什么喜欢
4. 文件与权限

### 字段建议

- 曲名
- 歌手
- 专辑
- 封面图链接
- 标签
- 一句短评
- 为什么喜欢
- 长说明
- 可见性
- 音乐文件
- 文件权限
- 是否允许下载

## 首页“声纹流”

首页中，音乐模块要以“声纹流”形式出现。

规则：

- 使用真实音乐封面
- 方形卡片
- 点击封面进入音乐详情
- 没封面时用首字兜底
- 不出现播放按钮

## 手机端规则

- 保持单列卡片
- 详情区的文件下载按钮必须易点
- 不出现复杂控制条
- 重点是“浏览、收藏、下载”，不是“播放”

## 第一版验收标准

- 页面第一眼像私人音乐收藏室，而不是播放器
- 能完成上传、查看、下载、编辑
- 音乐封面能进入首页声纹流
- 权限逻辑与书籍模块一致
