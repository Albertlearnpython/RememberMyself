# 景色模块手机上传与识别方案

## 这份文档解决什么

景色模块第一版不做旅游网站，也不做大而全的图库。

它先解决三件事：

1. 手机端可以直接上传照片
2. 尽量自动识别照片里的拍摄时间和地点
3. 上传后的景色可以进入首页“风景流”

## 使用场景

你的主要使用方式是：

- 苹果手机拍照
- 手机访问网站
- 直接上传照片
- 保存为一条景色记录
- 之后在景色页和首页都能看到

## 第一版范围

### 已纳入第一版

- 手机端多图上传
- 支持 `JPG / PNG / WEBP / HEIC / HEIF`
- 服务端自动转成网页可显示图片
- 读取 EXIF 拍摄时间
- 读取 EXIF GPS
- 如果识别到坐标，自动反查地点
- 景色列表页
- 景色详情区
- 首页“风景流”接入真实图片

### 暂不放进第一版

- 地图交互
- AI 自动命名
- 批量位置修正
- 上传前预览确认弹窗
- 多层标签系统

## 数据结构

### SceneryEntry

代表一次景色记忆，而不是一张孤立图片。

核心字段：

- 标题
- 一句说明
- 为什么记得这里
- 长说明
- 拍摄时间
- 纬度 / 经度
- 国家 / 省份 / 城市 / 区域 / 地点名
- 可见性

### SceneryPhoto

代表这次景色记录中的单张图片。

核心字段：

- 所属景色记录
- 转码后的图片
- 原始文件名
- 宽高
- 单图拍摄时间
- 单图坐标
- EXIF 原始解析结果
- 排序

## 自动识别规则

### 时间

优先读取：

- `DateTimeOriginal`
- `DateTimeDigitized`
- `DateTime`

如果图片里没有时区信息，第一版默认按网站当前时区处理。

### 地点

优先读取：

- `GPSLatitude`
- `GPSLongitude`

如果图片内带 GPS，第一版会尝试做一次反查，补出：

- 国家
- 省份
- 城市
- 区域
- 地点名
- 地址说明

### 失败策略

如果识别不到：

- 不阻塞保存
- 不弹重错误框
- 允许后续手动补字段

## 首页接入规则

首页风景流使用每条景色记录的首图作为流带封面。

流带卡片展示：

- 图片
- 标题
- 地点 / 日期

点击后进入景色详情页。

## 手机端交互原则

- 搜索和新增入口放在顶部
- 上传控件尽量少解释，直接可用
- 列表卡片优先看图
- 详情先看大图，再看地点和文字
- 编辑抽屉保持单列可滚动

## 当前实现口径

当前第一版采用：

**上传后保存，同时自动识别并尽量回填。**

也就是说，这一版先保证“能用、顺手、能真的留下来”。

如果后面要再升级，可以在此基础上补：

- 上传后识别预览页
- 确认后再入库
- 更细的地点纠错
- 地图卡与路线记忆
